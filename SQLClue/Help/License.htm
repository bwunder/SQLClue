Licensing SQLClue

Three license levels are available:

  1. Workstation Client - supports all SQLClue Configuration Compare capabilities and provides a contributor interface to a licensed Data Center Runbook located elsewhere on the LAN. No license is required for the SQLClue Workstation level.
  2. Local Server- the host SQL Server may be targeted for SQL Configuration Archive. The Data Center Runbook may be installed and accessed by any number of SQLClue Workstation Clients. The Automation Controller Windows service is available for use. All Configuration Compare capabilities are supported. 
  3. Environment Server - The SQL Configuration Archive can monitor any number of SQL Instances for changes. The Data Center Runbook may be installed and accessed by any number of SQLClue Workstation Clients. The Automation Controller Windows service is available for use. All Configuration Compare capabilities are supported.

Evaluation Server - SQLClue Evaluation Server may be installed and all capabilities may be used as unlicensed trial software for 14 days at no cost. Up to 3 SQL Servers may be targeted for SQL Configuration Archive automated change monitoring during the evaluation and the Data Center Runbook may be used. To enable Evaluation Server select install from the SQL Configuration Archive or the Data Center Runbook option of the SQLClue Workstation 'Tools' menu. There is a 7 day grace period after the 14 day trial period where SQLClue Server will continue to work after the trial period has expired. The SQLClue Administrator's Console will no longer load after the grace period has expired.

Principles

The SQL Server Host Key is automatically generated at the time the SQLClue Configuration Archive data store is initially configured. This key can be found at the top of of the Licensing form that is opened from the SQLClue 'Tools menu. The Host Key includes identifying information about the SQLClue Repository Host.  The SQL Server Host Key must be provided at the time a license is requested. It is used to generate the License Code. The Company Name and the desired license level (Local or Environment) must also be provided at the time of license request.

The date portion of the SQL Server Host Key is used to determine the locale of the host. If the host is moved to a different locale than indicated by the submitted Host Key, some SQLClue reports may no longer render properly.

A License Code will be provided  only enable SQLClue on the Windows Server and for the SQL Server Instance indicated by the SQL Server Host Key. Enter the License Code, the Company Name and the License Date into the Licensing Dialog exactly as shown on the License Certificate that has been sent to you.

To assure trouble free use of SQLClue, obtain a license code before making any changes to the licensing information. Apply change only as directed by the instructions provided with the license code. If any licensing information is inadvertantly change, be sure to remove it by selecting the [Reset] button on the Licensing dialog to revert to unlicensed mode. Closing the SQLClue Administrator's Consoleator with invalid licensing data entered in the License dialog will disable SQLClue.
