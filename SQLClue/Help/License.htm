On-site installation and training available. Happy to customize to your specifications. Please inquire.
 
Prerequisites for SQLClue include:
  • Microsoft Visual Studio 2010 ReportViewer
  • Microsoft .NET Framework 4.0
  • Microsoft SQL Client 11.0 Connectivity Components
  • Microsoft SQL Server 2012 for all SQLClue Server Components
  • TrueCrypt 7.1 for Encryption Hiearachy Administration
All are freely redistributable and available for download directly from the creator's web sites. Links can be found on the SQLClue download page.
 
Workstation is client software only - does not require a local SQL Server instance, only SQL Server Native Client 11. SQL Configuration & DDL Compare, the Runbook Peer, Report Server without the server' capabilities - but with only the default trace report available - and the ability to upgrade to Server are enabled in SQLClue Workstation.
SQLClue Server is required for components that need a database including:
  • SQL Configuration & DDL archive of any available user selected SQL Server 2000/2005/2008/2012 instance
  • Hosting a Runbook on the local network.
  • Encryption Hierarchy Administration. (requires an offsite hub database that can be hosted on SQL Azure or a user specified private cloud location.)
  • optional Automation Controller Windows Scheduler Service
1 The number of target SQL Servers that can be supported by a SQLClue host is limited by the feature capabilities of the targeted SQL Servers and the performance and throughput capabilities of the SQLClue host system platform. Server class hardware running Windows Server 2008 R2 with min 8GB RAM and high speed network bandwidth is recommended for any SQL Server that does real work. For evaluation purposes, any configuration that can support a SQL Server 2012 Developer Edition installation can be used.
